# 前端规范（对齐 TDesign 官方 Starter）

## 1. 设计基线

1. 以 `Tencent/tdesign-vue-next-starter` 作为视觉与交互基线，不自定义“风格化皮肤”。
2. 优先使用 TDesign 设计令牌（`--td-*`）与组件默认样式，不覆盖主色、字号体系、状态色语义。
3. 页面风格保持企业后台“克制、清晰、可维护”：禁止大面积渐变背景、夸张阴影、超大圆角。

## 2. 布局规范

1. 采用标准后台框架：`侧边导航 + 顶部栏 + 内容区`。
2. 侧边栏使用 TDesign dark menu 规范；顶部栏使用浅色容器与细边框分割。
3. 内容区统一 `24px` 内边距，卡片容器统一白底 + `1px` 边框 + `6px` 圆角。

## 3. 组件与交互规范

1. CRUD 页面结构固定：标题区、筛选工具栏、数据表格、分页、弹窗表单。
2. 表格、分页、按钮、标签、弹窗优先使用 TDesign 原生组件，不自建替代实现。
3. 状态展示统一语义：启用 `success`，禁用 `warning`，错误 `danger`。
4. 未经确认不新增动画效果；交互动效保持默认或极轻量。

## 4. 代码规范

1. 视觉样式集中在全局样式文件和少量页面级 class，避免散落的大量内联 style。
2. 业务逻辑与样式解耦：API 调用、状态管理、布局样式独立维护。
3. 新页面开发先复用现有模板页结构，再补业务字段和接口。

## 5. 验收标准

1. 与官方 Starter 对比，布局节奏、组件状态和信息层级无明显偏差。
2. 常见分辨率下（1440、1366、1200）无错位、无拥挤、无风格突变。
3. 每个 CRUD 页面具备一致操作路径：查、增、改、删、分页、状态反馈。
